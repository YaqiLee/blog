发表于 2020-12-03

元字符

普通元字符

字符 描述
\s 匹配任何空白字符，包括空格、制表符、换页符
\S 匹配任何非空白字符
\b 匹配单词边界
\B 匹配非单词边界
\t 匹配一个制表符
\d 匹配一个数字
\D 匹配一个非数字
\w 匹配一个数字、字母、下划线
\W 匹配一个非数字、字母、下划线
\f 匹配一个换页符
\n 匹配一个换行符
\r 匹配一个回车符
.(点) 匹配一个非换行的任意字符

限定元字符

字符 描述
* 匹配 0 次或多次
+ 匹配 1 次或多次
? 匹配 1 次或 1 次
^ 匹配输入字符串的开始位置
$ 匹配输入字符串的结束位置
x|y 匹配 x 或 y
{n} 匹配 n 次
{n,} 至少匹配 n 次 或多次
{n,m} 匹配 n 次 到 m 次
[a-z] 匹配 a-z 范围内的字符
[xyz] 匹配 包含的任一字符
^[xyz] 匹配 未包含的任一字符
(pattern) 匹配 并获取字符
(?:pattern) 匹配 pattern，但不返回结果
(?=pattern) 正向肯定定预查，不获取结果
(?!pattern) 正向否定预查，与肯定预查正好相反，不获取结果
(?<=pattern) 反向肯定预查
(?<!pattern) 反向否定预查

修饰符

字符 描述
i 不区分大小写
g 全局匹配
m 多行匹配
s 特殊字符圆点 . 包含换行符 \n

非贪婪匹配

在限定符后加?为非贪婪匹配，匹配长度尽可能少的匹配结果

字符 描述
*? 匹配 0 次或多次，但尽可能
+? 匹配 1 次或多次
?? 匹配 0 次或 1 次
{n,}? 至少匹配 n 次
{n,m}? 匹配 n 次到 m 次

*限定符， 匹配 0 次或多次

+限定符，匹配 1 次或多次

?限定符，匹配 0 次或 1 次

示例（js）

(pattern)：匹配 pattern，并且获取匹配结果

(?:pattern)

(?=pattern)：正向肯定预查

(?!pattern)：正向否定预查，与肯定预查正好相反

(?<=pattern): 反向肯定预查，与正向肯定预查，匹配方向相反

(?<!pattern): 反向否定预查

Web Components 是一套创建自定义可复用元素的技术。实现 web Compnent 主要用到三种技术：

  • Custom elements（自定义元素）：主要通过 window.customElements 的 API 创建自定义元素。例如： window.customElements.define('my-custom-element', MyCustomElement);
  • Shadow DOM：将通过 shadow dom 创建的元素私有化，与文档中其他元素样式与脚本的隔离，使其不会相互影响。
  • HTML templates（HTML 模板）：html 模板，通过标签 <template> 和<slot> 定义,见用法 2
